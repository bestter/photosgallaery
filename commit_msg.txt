🧪 Add edge case test for DeleteReport with non-existent ID

🎯 **What:**
Added a robust integration test for `AdminController.DeleteReport` when the requested report ID does not exist.

📊 **Coverage:**
The `DeleteReport_WithInvalidId_ReturnsNotFound` test now seeds the database with a valid report before attempting to delete a non-existent ID. It verifies that a `404 Not Found` response is returned and that the existing report in the database remains unaffected.

✨ **Result:**
Improved testing coverage for edge cases involving data deletion, ensuring safe failure behavior and preventing unintended side-effects when an invalid resource ID is provided.
