🎯 **What:** The moderation-service was susceptible to a decompression bomb attack where a tiny file payload could decode into massive RAM consumption. An explicit size limit has been configured using `Image.MAX_IMAGE_PIXELS` to prevent large images from being processed.
⚠️ **Risk:** The potential impact if left unfixed is severe - a malicious actor could send small malicious images that crash the service with Out-Of-Memory (OOM) errors, taking down the entire API via Denial-of-Service.
🛡️ **Solution:** The fix limits the max allowed pixels to 10,000,000 and adds an explicit try/catch block for `Image.DecompressionBombError` to immediately drop the request and return a 400 Bad Request instead of an unhandled 500 error. The unbounded `file.read()` issue highlighted in the prompt was already mitigated with a `while` loop that chunks memory locally.
